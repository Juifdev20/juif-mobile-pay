API Documentation

Intégrez facilement les paiements mobiles (M-Pesa, Airtel Money, Orange Money) dans vos applications avec notre API REST simple et puissante.

URL de Base

Base URL 
https://votre-domaine.com/api

Remplacez votre-domaine.com par votre domaine ou utilisez http://localhost:3000/api en développement local.

Authentification

Cette API est actuellement en mode développement et ne nécessite pas d'authentification. En production, nous recommandons d'ajouter une authentification par clé API ou JWT.

Endpoints

POST /api/initiate_pay

Initialise une nouvelle transaction de paiement mobile.

Paramètres du Corps (JSON)

Paramètre Type Requis Description
amount number Oui Montant du paiement en FC
phone string Oui Numéro de téléphone du client (10 chiffres minimum)
service string Oui Service de paiement: mpesa, airtel, ou orange

Exemple de Requête

cURL 
curl -X POST https://votre-domaine.com/api/initiate_pay \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 50000,
    "phone": "0828497218",
    "service": "mpesa"
  }'
JavaScript (fetch) 
const response = await fetch('https://votre-domaine.com/api/initiate_pay', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    amount: 50000,
    phone: '0828497218',
    service: 'mpesa'
  })
});

const data = await response.json();
console.log(data);

Réponse Succès (200)

{
  "success": true,
  "reference": "MPESA_20260424104100_f6k6n1mv",
  "message": "Transaction initiée avec succès",
  "transaction": {
    "reference": "MPESA_20260424104100_f6k6n1mv",
    "amount": 50000,
    "phone": "0828497218",
    "service": "mpesa",
    "status": "waiting",
    "created_at": "2026-04-24T10:41:00.513Z",
    "updated_at": "2026-04-24T10:41:00.513Z"
  }
}

Codes de Réponse

200 Succès 400 Paramètres invalides
GET /api/check_request

Vérifie le statut d'une transaction existante.

Paramètres de Requête

Paramètre Type Requis Description
reference string Oui Référence de la transaction retournée lors de l'initialisation

Exemple de Requête

cURL 
curl -X GET "https://votre-domaine.com/api/check_request?reference=MPESA_20260424104100_f6k6n1mv"

Réponse Succès (200)

{
  "success": true,
  "transaction": {
    "reference": "MPESA_20260424104100_f6k6n1mv",
    "amount": 50000,
    "phone": "0828497218",
    "service": "mpesa",
    "status": "SUCCESS",
    "created_at": "2026-04-24T10:41:00.513Z",
    "updated_at": "2026-04-24T10:41:14.982Z"
  },
  "status": "SUCCESS"
}

Statuts Possibles

  • waiting - En attente de confirmation du client
  • SUCCESS - Paiement réussi
  • CANCELLED - Paiement annulé par le client

Codes de Réponse

200 Succès 400 Paramètre manquant 404 Transaction non trouvée
POST /api/update_status

Met à jour le statut d'une transaction (utilisé par le simulateur).

Paramètres du Corps (JSON)

Paramètre Type Requis Description
reference string Oui Référence de la transaction
status string Oui Nouveau statut: SUCCESS ou CANCELLED

Exemple de Requête

cURL 
curl -X POST https://votre-domaine.com/api/update_status \
  -H "Content-Type: application/json" \
  -d '{
    "reference": "MPESA_20260424104100_f6k6n1mv",
    "status": "SUCCESS"
  }'

Réponse Succès (200)

{
  "success": true,
  "message": "Statut mis à jour avec succès",
  "reference": "MPESA_20260424104100_f6k6n1mv",
  "status": "SUCCESS"
}

Codes de Réponse

200 Succès 400 Paramètres invalides 404 Transaction non trouvée
GET /api/transactions

Récupère toutes les transactions enregistrées.

Exemple de Requête

cURL 
curl -X GET https://votre-domaine.com/api/transactions

Réponse Succès (200)

{
  "success": true,
  "transactions": [
    {
      "reference": "MPESA_20260424104100_f6k6n1mv",
      "amount": 50000,
      "phone": "0828497218",
      "service": "mpesa",
      "status": "SUCCESS",
      "created_at": "2026-04-24T10:41:00.513Z",
      "updated_at": "2026-04-24T10:41:14.982Z"
    }
  ]
}

Codes de Réponse

200 Succès
DELETE /api/transactions

Supprime toutes les transactions (reset de la base de données).

Exemple de Requête

cURL 
curl -X DELETE https://votre-domaine.com/api/transactions

Réponse Succès (200)

{
  "success": true,
  "message": "Transactions réinitialisées"
}

Codes de Réponse

Limites de Taux

Gestion des Erreurs

Toutes les erreurs retournent un objet JSON avec le format suivant : 

{
  "success": false,
  "message": "Description de l'erreur"
}
